# 代码节点出入参说明：按逻辑合并审核结果

对应代码：`代码节点.py`

## 这个节点做什么

根据 `logicTopology` 的 AND / OR 关系合并所有规则结果，得到最终审核结果，并输出每条规则的审核明细。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ruleResults` | Array、Object 或 JSON字符串 | 是 | 迭代子工作流返回并聚合后的规则结果。 |
| `logicTopology` | Object 或 JSON字符串 | 是 | 规则组合逻辑树。腾讯平台建议传 `节点1.Output.logicTopologyStr`，类型选 `str`。 |

### `ruleResults` 推荐层级

```text
ruleResults: Array
  - item: Object
      ruleResult: Array
        - item: Object
            ruleCode: String
            ruleContent: String
            ruleResult: String，"通过" 或 "不通过"
            suspicionList: Array，可选
      reasoningContent: String
      extractionList: Array<Object>
```

也兼容腾讯子工作流结束节点包装后的结构：

```text
ruleResults: Array
  - item: Object
      Output: Object
        one_rule_result: Array
          - item: Object
              ruleResult: Array
              reasoningContent: String
              extractionList: Array<Object>
```

也兼容单条规则直接传：

```text
ruleResults: Array
  - item: Object
      ruleCode: String
      ruleContent: String
      ruleResult: String
      reasoningContent: String
      ruleKeywordGuide: Array<Object>
      suspicionList: Array，可选
```

### `logicTopology` 层级

```text
logicTopology: Object
  type: String，"GROUP" 或 "RULE_REF"
  operator: String，仅 GROUP 有，"AND" 或 "OR"
  children: Array<Object>，仅 GROUP 有
  ruleCode: String，仅 RULE_REF 有
```

## 腾讯平台里 `logicTopology` 怎么配

推荐配置：

```text
变量名：logicTopology
数据来源：引用
绑定：节点1.Output.logicTopologyStr
类型：str
```

原因：`logicTopology` 是递归树对象，腾讯 Output schema 容易把深层 `children` 裁剪掉。传 `logicTopologyStr` 字符串后，本代码节点会自动解析回完整对象。

## 联调踩坑记录

1. 合并审核入参里的 `logicTopology` 必须是完整 OR/AND 树。
   如果看到 `children` 里只有 `{ "type": "GROUP", "ruleCode": "" }`，说明传的是被裁剪后的对象，要改用 `logicTopologyStr`。

2. `ruleResults` 可能被腾讯子工作流包成 `Output.one_rule_result` 或 `one_rule_result`。
   节点3代码已兼容拆包；如果读不到规则，优先检查平台代码是否同步了这版。

3. `suspicionList` 要配置在 `Output.ruleResults[].suspicionList`。
   不要配置到 `Output.suspicionList`。配错层级时会看到最外层 `suspicionList: null`，每条规则里反而没有原因。

4. `suspicionList` 类型必须是 `[obj]`。
   如果输出变成字符串，说明 schema 类型配成了 `str` 或上游结构化输出类型不对。

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `finalResult` | String | 最终审核结果：`"通过"` 或 `"不通过"`。 |
| `ruleResults` | Array<Object> | 每条规则的审核明细。 |
