# 代码节点出入参说明：精解结果结构化

对应代码：`代码-将精解结果结构化.py`

## 这个节点做什么

把“精解 LLM”输出的 `extraction_data` 转成数组，并按 `keywordCode` 排序，输出 `extractionList`。

同时会做一层字段兜底：

```text
results[].materialSource 缺失时补为空字符串
```

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `extraction_data` | String 或 Array | 建议是 | 绑定“1精解 LLM”的输出文本。为空时输出空数组；如果是字符串，必须是合法 JSON 数组字符串。 |

### `extraction_data` 层级

```text
extraction_data: Array
  - item: Object
      keywordCode: String
      found: Boolean
      results: Array
        - item: Object
            materialId: String
            materialName: String
            materialSource: String，可选
            rawText: String
            value: String 或 Number 或 Boolean
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `extractionList` | Array<Object> | 排序后的精解结果数组。 |

## 可直接粘贴测试的入参示例

```json
{
  "extraction_data": "[{\"keywordCode\":\"1002001001\",\"found\":true,\"results\":[{\"materialId\":\"2018496043368521728\",\"materialName\":\"住院病案首页\",\"materialSource\":\"济南市医保局\",\"rawText\":\"无糖尿病史。\",\"value\":\"未确诊\"}]}]"
}
```

## 腾讯平台里怎么配

```text
变量名：extraction_data
数据来源：引用
绑定：1精解 LLM 节点的 Output
类型：str
```

如果这里没绑定到 1精解 LLM 的 Output，出参就会变成：

```json
{
  "extractionList": []
}
```

后面的“逐条认定”就会因为没有结构化审核证据，判成“信息缺失”。

## 联调踩坑记录

1. `extraction_data` 要绑定 `原文精解.Output.extraction_data`。
   不要绑定整个 `Output`，也不要绑定 `results.value` 之类的子字段。

2. 精解结构化节点不负责把 `refContent` 变成 `rawText`。
   业务上 `rawText` 是精解阶段字段，应该由原文精解 LLM 直接输出。

3. 如果本节点输出 `extractionList: []`，先看本节点实际入参。
   入参里 `extraction_data` 为空是绑定问题；入参有数组但出参为空才是代码问题。

## 对应出参示例

```json
{
  "extractionList": [
    {
      "keywordCode": "1002001001",
      "found": true,
      "results": [
        {
          "materialId": "2018496043368521728",
          "materialName": "住院病案首页",
          "materialSource": "济南市医保局",
          "rawText": "无糖尿病史。",
          "refContent": "无糖尿病史。",
          "value": "未确诊"
        }
      ]
    }
  ]
}
```
