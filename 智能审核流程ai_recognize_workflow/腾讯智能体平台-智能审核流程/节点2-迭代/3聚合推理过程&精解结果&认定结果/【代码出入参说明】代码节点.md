# 代码节点出入参说明：迭代内出参聚合

对应代码：`代码节点.py`

## 这个节点做什么

把一次迭代里的三类结果合并到一个对象里：单条审核结果、推理过程、精解结果。输出最外层固定是 `output` 数组。

注意这里会同时保留两类证据字段：

```text
extractionList.results[].rawText
  精解阶段的原始证据片段

ruleResult[].suspicionList[].sources[].refContent
  审核结论阶段的引用证据
```

两者业务含义不同，不要在聚合节点里互相改写。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ruleResult` | Array、Object 或 JSON字符串 | 否 | 来自“单条标准审核结果结构化”节点。 |
| `reasoningContent` | String、Object 或 JSON字符串 | 否 | 来自“提取推理过程”节点。 |
| `extractionList` | Array、Object 或 JSON字符串 | 否 | 来自“精解结果结构化”节点。 |

### 入参推荐层级

```text
ruleResult: Array
  - item: Object
      ruleCode: String
      ruleContent: String
      ruleResult: String
      suspicionList: Array，可选

reasoningContent: String

extractionList: Array
  - item: Object
      keywordCode: String
      found: Boolean
      results: Array<Object>，内部使用 rawText 表示精解阶段原始证据
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `output` | Array<Object> | 固定只有 1 个对象，用于给父流程聚合。 |

## 联调踩坑记录

1. 聚合节点输出固定是 `output: [obj]`。
   腾讯子工作流结束节点里应配置 `Output.one_rule_result: [obj]`，并绑定这里的 `output`。

2. 聚合结果会同时包含精解证据和审核结论证据。
   `extractionList.results[].rawText` 保留精解原文证据；`ruleResult[].suspicionList[].sources[].refContent` 保留审核结论引用证据。
