# 代码节点出入参说明：规则库转可迭代数组

对应代码：`节点1代码.py`

## 这个节点做什么

把完整认定标准里的 `ruleRepository` 拆成腾讯平台迭代可用的 `rulesArray`，同时带出病种信息和逻辑拓扑。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `certification_list` | Object、Array 或 JSON字符串 | 是 | 完整认定标准。可以直接传对象，也可以传 JSON 字符串；如果是数组，取第 1 个对象。 |
| `material_list` | String | 否 | 患者材料列表。本节点不使用，只是兼容流程入参。 |

### `certification_list` 层级

```text
certification_list: Object 或 Array<Object>
  meta: Object
    chronicDiseaseCode: String
    chronicDiseaseName: String
  ruleRepository: Array
    - item: Object
        ruleCode: String
        ruleContent: String
        experience: String 或 Null
        ruleKeywordGuide: Array
          - item: Object
              keywordCode: String
              keywordContent: String
              dataType: String
              enumOptions: Array<String>，可选
              required: Boolean
              instruction: String，可选
  logicTopology: Object
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `chronicDiseaseCode` | String | 病种编码。 |
| `chronicDiseaseName` | String | 病种名称。 |
| `rulesArray` | Array<Object> | 可迭代规则数组，按 `ruleCode` 排序。 |
| `rulesCount` | Number | 规则数量。 |
| `logicTopology` | Object | 原样输出规则组合逻辑树。 |
| `logicTopologyStr` | String | 规则组合逻辑树的 JSON 字符串。腾讯平台传递递归对象可能裁剪层级，节点3建议绑定这个字段。 |

## 腾讯平台配置提醒

如果后面的“合并审核”节点需要使用逻辑拓扑，建议绑定：

```text
logicTopology = 节点1.Output.logicTopologyStr
类型 = str
```

不建议在腾讯平台里继续传 `logicTopology` 对象本身，因为它是递归树结构，平台的 Output schema 容易裁剪深层 `children`。

## 联调踩坑记录

1. `logicTopology` 是递归树，腾讯 Output schema 容易裁剪深层 `children`。
   典型错误入参会变成只有两个空 `GROUP`，没有里面的 `operator/children`。

2. 为避免裁剪，节点1额外输出 `logicTopologyStr`。
   后续合并审核节点里，变量名仍叫 `logicTopology`，但类型选 `str`，绑定 `节点1.Output.logicTopologyStr`。
