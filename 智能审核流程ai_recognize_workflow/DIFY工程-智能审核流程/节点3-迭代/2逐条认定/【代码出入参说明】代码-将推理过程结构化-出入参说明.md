# 代码节点出入参说明：推理过程结构化

对应代码：`代码-将推理过程结构化.py`

## 这个节点做什么

把模型输出的推理过程整理成字符串。它不解析推理内容，只负责兼容不同入参形式并稳定输出 `reasoningContent`。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `reasoningContent` | String、Object 或 JSON字符串 | 否 | 推理过程文本。 |
| `input.reasoningContent` | String | 否 | 兼容嵌套入参；当直接入参为空时读取。 |

### 可接受层级

```text
reasoningContent: String
```

或：

```text
reasoningContent: Object
  reasoningContent: String
```

或：

```text
input: Object
  reasoningContent: String
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `reasoningContent` | String | 推理过程文本。为空时输出空字符串。 |

### 出参层级

```text
Object
  reasoningContent: String
```

## 可直接粘贴测试的入参示例

```json
{
  "reasoningContent": "规则10001判断：材料中出现慢性肾衰竭和尿毒症期描述，因此该规则通过。"
}
```

## 对应出参示例

```json
{
  "reasoningContent": "规则10001判断：材料中出现慢性肾衰竭和尿毒症期描述，因此该规则通过。"
}
```
