# 代码节点出入参说明：单条标准审核结果结构化

对应代码：`代码-单条标准审核结果结构化.py`

## 这个节点做什么

把“逐条认定”模型输出整理成统一的 `ruleResult` 数组。通过时只保留规则结果；不通过时会补充疑点列表 `suspicionList`。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ruleResult` | Object、Array、JSON字符串 | 否 | 逐条认定模型输出。 |
| `text` | String | 否 | 兼容字段；当 `ruleResult` 为空时，会尝试读取 `text`。 |
| `items` | Object | 否 | 兜底规则信息，常用于补充 `ruleCode`、`ruleContent`。 |

### `ruleResult` 推荐层级

```text
ruleResult: Object 或 Array
  ruleCode: String，规则编码
  ruleContent: String，规则原文
  ruleResult: String，"通过" 或 "不通过"
  suspicionList: Array，仅不通过时需要
    - item: Object
        suspicionType: String，疑点类型
        detail: String，疑点说明
        sources: Array，证据来源
          - item: Object 或 String
              materialId: String，可选，材料 ID
              materialName: String，可选，材料名称
              refContent: String，证据原文
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `ruleResult` | Array<Object> | 规则审核结果数组。即使只有 1 条规则，也会包成数组。 |

### 出参层级

```text
Object
  ruleResult: Array
    - item: Object
        ruleCode: String，可选
        ruleContent: String，可选
        ruleResult: String
        suspicionList: Array，仅 ruleResult="不通过" 时输出
          - item: Object
              suspicionType: String
              detail: String
              sources: Array
                - item: Object
                    materialId: String，可选
                    materialName: String，可选
                    refContent: String
```

## 可直接粘贴测试的入参示例

```json
{
  "ruleResult": {
    "ruleCode": "10001",
    "ruleContent": "各种原因造成慢性肾脏损伤，并出现肾功能异常达到尿毒症期",
    "ruleResult": "不通过",
    "suspicionList": [
      {
        "suspicionType": "证据不足",
        "detail": "材料中未找到明确 CKD5 期或尿毒症期描述。",
        "sources": [
          {
            "materialId": "MAT-CYQ-01",
            "materialName": "病案首页（住院基本信息）",
            "refContent": "主要诊断：慢性肾衰竭"
          }
        ]
      }
    ]
  }
}
```
