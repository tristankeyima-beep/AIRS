# 代码节点出入参说明：迭代内出参聚合

对应代码：`代码-出参聚合.py`

## 这个节点做什么

把一次规则迭代里的三个结果合并到一个对象中：单条审核结果、推理过程、精解结果。为了适配 DIFY 迭代汇总，最外层输出为 `output` 数组。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ruleResult` | Array、Object 或 JSON字符串 | 否 | 来自“单条标准审核结果结构化”节点。 |
| `reasoningContent` | String、Object 或 JSON字符串 | 否 | 来自“推理过程结构化”节点。 |
| `extractionList` | Array、Object 或 JSON字符串 | 否 | 来自“精解结果结构化”节点。 |

### 入参推荐层级

```text
ruleResult: Array
  - item: Object
      ruleCode: String
      ruleContent: String
      ruleResult: String
      suspicionList: Array，可选

reasoningContent: String

extractionList: Array
  - item: Object
      keywordCode: String
      其他字段: 任意类型
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `output` | Array<Object> | 固定只有 1 个对象，用于给 DIFY 迭代节点聚合。 |

### 出参层级

```text
Object
  output: Array
    - item: Object
        ruleResult: Array<Object>
        reasoningContent: String
        extractionList: Array<Object>
```

## 可直接粘贴测试的入参示例

```json
{

    {
      "ruleCode": "10001",
      "ruleContent": "各种原因造成慢性肾脏损伤，并出现肾功能异常达到尿毒症期",

    }

  "reasoningContent": "材料中出现慢性肾衰竭和尿毒症期描述，因此规则10001通过。",
  "extractionList": [
    {
      "keywordCode": "10001001",

        {
          "value": "慢性肾衰竭",

        }

}
```
