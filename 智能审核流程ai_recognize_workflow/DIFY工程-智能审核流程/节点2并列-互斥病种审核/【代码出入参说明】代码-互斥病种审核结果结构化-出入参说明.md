# 代码节点出入参说明：互斥病种审核结果结构化

对应代码：`代码-互斥病种审核结果结构化.py`

## 这个节点做什么

把大模型输出的互斥病种审核结果整理成稳定 JSON。模型输出可以是 JSON 字符串、Markdown 代码块里的 JSON，或已经解析好的对象。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `mutexReviewResult` | Object 或 JSON字符串 | 否 | 互斥病种审核模型的输出。为空时默认按“通过”处理。 |
| `text` | String | 否 | 兼容字段；当 `mutexReviewResult` 为空时，会尝试读取 `text`。 |

### `mutexReviewResult` 推荐层级

```text
mutexReviewResult: Object
  reviewResult: String，"通过" 或 "不通过"
  mutexDiseasesName: Array<String> 或 String，命中的互斥病种名称
  reason: String，原因说明
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `reviewResult` | String | 只会输出 `"通过"` 或 `"不通过"`。 |
| `mutexDiseasesName` | Array<String> | 命中的互斥病种名称，已去重。 |
| `reason` | String | 审核原因。为空时会自动补默认原因。 |

### 出参层级

```text
Object
  reviewResult: String
  mutexDiseasesName: Array<String>
  reason: String
```

## 可直接粘贴测试的入参示例

```json
{
  "mutexReviewResult": {
    "reviewResult": "不通过",
    "mutexDiseasesName": ["糖尿病", "糖尿病"],
    "reason": "患者材料中出现糖尿病诊断，命中互斥病种。"
  }
}
```

## 对应出参示例

```json
{
  "reviewResult": "不通过",
  "mutexDiseasesName": ["糖尿病"],
  "reason": "患者材料中出现糖尿病诊断，命中互斥病种。"
}
```
