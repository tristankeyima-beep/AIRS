# 代码节点出入参说明：规则库转可迭代数组

对应代码：`节点1代码.py`

## 这个节点做什么

把完整认定标准里的 `ruleRepository` 拆成 DIFY 迭代节点可用的 `rulesArray`，同时带出病种信息和逻辑拓扑。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `certification_list` | JSON字符串 | 是 | 完整认定标准。可以是一个对象，也可以是对象数组；如果是数组，取第 1 个对象。 |
| `material_list` | String | 否 | 患者材料列表。本代码节点没有使用，只是兼容工作流入参。 |

### `certification_list` 层级

```text
certification_list: Object 或 Array<Object>
  meta: Object
    chronicDiseaseCode: String，病种编码
    chronicDiseaseName: String，病种名称
  ruleRepository: Array
    - item: Object
        ruleCode: String，规则编码
        ruleContent: String，规则原文
        experience: String 或 Null，可选，经验判断说明
        ruleKeywordGuide: Array，提取项说明
          - item: Object
              keywordCode: String，提取项编码
              keywordContent: String，提取项说明
              dataType: String，字段类型，如 string、enum
              enumOptions: Array<String>，枚举值，dataType=enum 时常见
              required: Boolean，是否必填
  logicTopology: Object，规则组合逻辑树
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `chronicDiseaseCode` | String | 病种编码，来自 `meta.chronicDiseaseCode`。 |
| `chronicDiseaseName` | String | 病种名称，来自 `meta.chronicDiseaseName`。 |
| `rulesArray` | Array<Object> | 可迭代规则数组，按 `ruleCode` 排序。 |
| `rulesCount` | Number | 规则数量。 |
| `logicTopology` | Object | 原样输出的规则组合逻辑树。 |

### `rulesArray` 层级

```text
rulesArray: Array
  - item: Object
      ruleCode: String
      ruleContent: String
      experience: String 或 Null
      ruleKeywordGuide: Array<Object>
```

### `logicTopology` 层级

```text
logicTopology: Object
  type: String，"GROUP" 或 "RULE_REF"
  operator: String，"AND" 或 "OR"，仅 GROUP 有
  children: Array<Object>，仅 GROUP 有，可继续嵌套 GROUP 或 RULE_REF
  ruleCode: String，仅 RULE_REF 有，指向 ruleRepository 中的规则
```
