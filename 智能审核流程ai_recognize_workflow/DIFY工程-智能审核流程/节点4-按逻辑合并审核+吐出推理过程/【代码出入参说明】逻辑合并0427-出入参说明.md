# 代码节点出入参说明：按逻辑合并审核结果

对应代码：`逻辑合并0427.py`

## 这个节点做什么

根据 `logicTopology` 的 AND / OR 关系，合并所有单条规则的审核结果，得到最终审核结果。同时把每条规则的结果、精解结果和推理过程整理到最终出参里。若互斥病种审核为“不通过”，最终结果直接变为“不通过”。

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `ruleResults` | Array | 是 | DIFY 迭代聚合后的规则结果。通常来自“迭代内出参聚合”节点的 `output` 汇总。 |
| `logicTopology` | Object | 是 | 规则组合逻辑树。 |
| `mutexReviewResult` | Object 或 JSON字符串 | 否 | 互斥病种审核结果；如果为“不通过”，最终结果强制“不通过”。 |

### `ruleResults` 层级

```text
ruleResults: Array
  - item: Object
      ruleResult: Array
        - item: Object
            ruleCode: String
            ruleContent: String
            ruleResult: String，"通过" 或 "不通过"；"不符合" 会被转成 "不通过"
            suspicionList: Array，可选，仅不通过时常见
      reasoningContent: String
      extractionList: Array
        - item: Object
            keywordCode: String
            其他字段: 任意类型
```

### `logicTopology` 层级

```text
logicTopology: Object
  type: String
    - "GROUP": 逻辑组
    - "RULE_REF": 规则引用
  operator: String，仅 GROUP 有，"AND" 或 "OR"
  children: Array<Object>，仅 GROUP 有，可继续嵌套
  ruleCode: String，仅 RULE_REF 有，对应单条规则的 ruleCode
```

### `mutexReviewResult` 层级

```text
mutexReviewResult: Object
  reviewResult: String，"通过" 或 "不通过"
  mutexDiseasesName: Array<String>
  reason: String
```

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `finalResult` | String | 最终审核结果：`"通过"` 或 `"不通过"`。 |
| `ruleResults` | Array<Object> | 每条规则的审核明细，按迭代出现顺序输出。 |

### `ruleResults` 出参层级

```text
Object
  finalResult: String
  ruleResults: Array
    - item: Object
        ruleCode: String
        ruleContent: String
        ruleKeywordGuide: Array<Object>，实际放的是 extractionList 精解结果
        reasoningContent: String
        ruleResult: String
        suspicionList: Array，可选，仅该规则不通过且有疑点时输出
          - item: Object
              suspicionType: String
              detail: String
              sources: Array
```

## 判断规则

```text
GROUP + AND：所有 children 都通过，才通过
GROUP + OR：任一 child 通过，就通过
RULE_REF：找到对应 ruleCode 的单条规则结果，ruleResult="通过" 才算通过
互斥病种：mutexReviewResult.reviewResult="不通过" 时，finalResult 直接为 "不通过"
```
