# 腾讯智能体平台搭建说明：图片识别流程

本目录用于在腾讯智能体平台复刻 `DIFY工程-图片识别流程` 工作流。

## 一、工作流目标

对医疗记录图片进行文字识别，按原图可见内容原样输出。

这个流程只做图像内容转写，不做诊断判断、不做字段归纳、不补全模糊内容。

## 二、主工作流入参

建议在腾讯平台启动节点配置：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `image` | image/file 或平台图片变量 | 是 | 待识别的医疗记录图片。 |
| `userRequirement` | str | 否 | 用户额外要求。没有额外要求时传空字符串。 |

如果腾讯平台当前应用使用的是批量图片变量，也可以命名为 `image_list`。后续 LLM 节点中只要绑定到实际图片输入即可。

## 三、节点顺序

```text
启动节点
  -> 节点1：图片识别（多模态 LLM）
  -> 结束节点
```

## 四、节点配置文件

| 节点 | 类型 | 配置说明 |
| --- | --- | --- |
| 节点1-图片识别 | 多模态 LLM | `节点1-图片识别/【LLM节点配置说明】图片识别.md` |

## 五、变量绑定

节点1输入：

```text
image = 启动节点.image
userRequirement = 启动节点.userRequirement
```

结束节点输出：

| 字段 | 类型 | 绑定 |
| --- | --- | --- |
| `recognizedText` | str | 节点1.Output |

## 六、联调注意

1. LLM 节点必须选择支持图片输入的模型。
2. 图片变量要作为“图片/文件输入”绑定，不要只把图片 URL 当普通文本塞进提示词。
3. 输出要求是原样转写，不要让模型自行新增标题、摘要、表格或结构化字段。
4. 对模糊、遮挡、缺失内容，按提示词要求输出 `[模糊]`、`无记录` 或 `[图像不完整]`。
