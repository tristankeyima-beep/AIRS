# LLM 节点配置说明：图片识别

参考 DIFY 原始提示词：

`DIFY工程-图片识别流程/【prompt】图片识别提示词`

## 一、这个节点做什么

识别医疗记录图片中的可见文字，并按照原图格式原样输出。

本节点不做医学推断、不做归纳总结、不新增字段。

## 二、模型要求

选择支持图片输入的多模态模型。

如果平台提供“视觉理解”“图片理解”“多模态 LLM”等节点类型，优先使用该类节点。

## 三、输入变量

| 变量名 | 类型 | 绑定来源 |
| --- | --- | --- |
| `image` | image/file | 启动节点.image |
| `userRequirement` | str | 启动节点.userRequirement，可为空 |

## 四、提示词

把下面内容放到腾讯多模态 LLM 节点提示词里，并把 `image` 变量绑定到该节点的图片输入区。

```text
### 图像关键信息识别提示词（医疗记录场景专用）

目标：确保从图像中提取关键信息时，100%基于图像可见内容，无添加、无遗漏、无推测，避免因图像质量、格式复杂性导致的识别错误。

一、核心原则（必须严格遵守）

1. 零推测原则
   - 仅提取图像中清晰可见且完整的文字/数字，禁止基于医学知识或上下文进行任何推断。
2. 事实锚定原则
   - 所有信息必须直接对应图像位置，禁止整合外部知识。
3. 无修饰原则
   - 禁止添加解释性语言、医学术语扩展或格式美化。

二、关键识别要点（分步执行）

步骤1：图像预处理与范围界定

- 必须验证：仅处理用户指定的图像区域，忽略系统界面元素、任务栏、水印、页码等非医疗主体内容。
- 必须检查：图像是否完整，如表格是否被截断。若关键区域缺失，标记为“[图像不完整]”。

步骤2：结构化信息提取（按字段优先级）

1. 身份标识
   - 仅提取姓名、病案号、性别、年龄等图像明确出现的信息。
   - 禁止从模糊文字中猜测或补全姓名。

2. 时间数据
   - 严格按图像格式记录。
   - 禁止转换时区、日期格式或推断时间含义。

3. 诊断信息
   - 区分“入院诊断”“出院诊断”“主要诊断”等标题。
   - 记录疾病编码。
   - 若诊断描述含括号注释，完整保留括号内容。
   - 禁止合并或归并相似诊断。

4. 检查数值
   - 表格数据需按行列对应。
   - 记录图像中明确出现的单位。
   - 参考值仅提取图像明确标注部分。
   - 禁止补充缺失单位或推断量纲。

5. 文本描述
   - 逐字转录。
   - 保留标点符号和换行结构。
   - 模糊文字用“[模糊]”标注。
   - 禁止修正语法错误或简化语句。

步骤3：冲突与缺失处理

- 多页冲突：同一信息在不同页面矛盾时，分别列出并标注页面位置。
- 数据缺失：字段存在但无内容时，按原字段名称标注“无记录”。
- 无法识别：文字模糊或遮挡时，标注“图像模糊，无法识别”或在对应位置标注“[模糊]”。

三、高频问题规避清单

1. 表格跨页断裂：检查表头与内容是否匹配。若断裂则标注“[表格不完整]”，不强行拼接。
2. 手写体/低分辨率文字：仅提取可辨认字符，其余标记“[模糊]”。
3. 重复信息：仅记录首次出现位置，不重复输出。
4. 干扰元素：完全忽略非医疗内容。
5. 医学术语缩写：按图像原样记录，不扩展、不解释。

四、输出规范（强制执行）

只输出原文与原格式：

1. 严禁自建结构：不得新增标题、段落、列表、编号、表格、总结区块或任何“说明/执行情况/校验结果”。
2. 原样复刻：严格按照原图的标题层级、段落、表格、行列顺序、换行与标点输出。
3. 不增不减字段：仅输出原图中出现的字段与内容，顺序与原图一致。
4. 缺失与模糊标注规则：
   - 若原图已有对应字段位置但内容空缺，用“无记录”替代空缺内容。
   - 若内容模糊无法辨认，用“[模糊]”替代无法辨认部分。
5. 位置标注：默认不添加任何额外标注；仅在用户明确要求时，才在不破坏原图格式的前提下添加位置标注。

禁止项：

- 任何总结性语句
- 任何未在原图中出现的内容
- 任何主观评价或格式美化

最终校验要求：

输出前必须对照原始图像逐行核对，确保无数字/字母/标点错误，无添加外部知识，输出结构与原图格式严格一致，未新增任何标题、段落或说明性内容。

若存在任何不确定性，宁可标记“[模糊]”，不可猜测。

用户额外要求：
【在这里用腾讯变量选择器插入：userRequirement】
```

## 五、输出

LLM 节点输出直接作为最终识别结果。

结束节点建议配置：

```text
recognizedText = 图片识别 LLM 节点输出文本
类型 = str
```

## 六、联调注意

1. 图片必须绑定到模型的图片输入区，不要只在提示词里写图片变量名。
2. 如果模型输出了“以下是识别结果”等说明文字，回到提示词加强“只输出原文与原格式”。
3. 如果表格被模型重排，建议降低自由发挥程度，并强调“按原图行列顺序输出”。
4. 如果图片太模糊，允许输出 `[模糊]`，不要让模型猜测。
