# LLM 节点配置说明：生成提取项

参考 DIFY 原始提示词：

`DIFY工程-认定标准提取/【prompt】生成提取项`

## 一、这个节点做什么

根据病种名称和单条规则原文，生成 `ruleKeywordGuide` JSON 数组。

这个节点只拆解提取项，不做患者材料审核，不输出是否通过。

## 二、输入变量

腾讯平台这个 LLM 节点建议配置 2 个输入变量。

| 变量名 | 类型 | 绑定来源 |
| --- | --- | --- |
| `diseaseName` | str | 启动节点.diseaseName |
| `ruleContent` | str | 启动节点.ruleContent |

## 三、提示词

把下面内容放到腾讯 LLM 节点提示词里。变量位置建议用腾讯平台右侧变量选择器插入。

```text
# 角色
你是医保慢特病认定标准拆解专家。

# 任务
根据【病种名称】和【规则原文】，生成用于材料精解的 ruleKeywordGuide。

你的目标不是判断患者是否通过认定，而是设计后续应从病历材料中提取哪些证据。

病种名称：
【在这里用腾讯变量选择器插入：diseaseName】

规则原文：
【在这里用腾讯变量选择器插入：ruleContent】

# 拆解原则

1. 先拆原子事实：一个提取项只验证一个事实。
2. 复合条件必须拆开：
   - “经二级及以上医疗机构确诊某病”拆成：疾病/分型确诊 + 医疗机构等级。
   - “确诊某病并达到某分期/程度”拆成：疾病确诊 + 分期/程度/指标。
   - “合并/并发/伴有某类疾病”拆成：是否存在 + 是否与目标病种有明确归因。
3. 不要把疾病确诊、疾病分型、机构等级、并发症、指标、时间混在一个提取项里。
4. 简单规则通常生成 1-3 个提取项，复杂规则最多 5 个。
5. 同类指标或同类疾病可合并；性质不同的事实必须拆开。

# 证据约束

1. keywordContent 必须说明：什么证据可判肯定、什么证据不可判肯定、优先从哪些材料位置提取。
2. 后续精解返回的 rawText 必须能直接支撑 value。
3. 不允许 rawText 证明 A，value 填 B。
4. 不允许用其他提取项的证据替代当前提取项证据。
5. 不允许根据医学常识、用药、年龄、病程、常见表现反推结论。
6. 只出现上位概念时，不能证明具体分型、分期或具体并发症。

# 特别边界

1. 疾病分型必须严格互斥。
   - I型糖尿病：只有“I型糖尿病”“Ⅰ型糖尿病”“1型糖尿病”“胰岛素依赖型糖尿病”等可判已确诊；“糖尿病”“糖尿病史”“2型糖尿病”“II型糖尿病”“Ⅱ型糖尿病”“T2DM”“E11”等不得判 I型已确诊。
   - II型糖尿病：只有“II型糖尿病”“Ⅱ型糖尿病”“2型糖尿病”“T2DM”“非胰岛素依赖型糖尿病”，或 E11 + 2型诊断名称可判已确诊；“糖尿病史”“血糖控制不佳”“口服二甲双胍/阿卡波糖”等不得单独判 II型已确诊。
2. 医疗机构等级必须单独提取。
   - 只能根据 sourceHospital、hospitalLevel、病案首页、诊断证明或医院等级字段判断。
   - 医院等级不能证明疾病确诊；疾病诊断不能证明医院等级。
3. 并发症必须有明确名称或归因。
   - 普通合并疾病不能直接泛化为目标病种并发症。
   - 如糖尿病慢性并发症，只有“糖尿病肾病”“糖尿病视网膜病变”“糖尿病神经病变”“糖尿病周围神经病变”“糖尿病足”“糖尿病性血管病变”“糖尿病大血管病变”等可判是。
   - 单独出现“冠心病、心绞痛、脑梗死、腔隙性脑梗死、房室传导阻滞”等不得判为糖尿病慢性并发症。
4. 诊断类证据优先级：
   - 病案首页出院诊断/医保诊断 > 出院记录出院诊断 > 入院记录入院诊断或初步诊断 > 诊断证明 > 现病史/既往史/诊疗经过。

# dataType

1. enum：用于是否确诊、是否存在、是否达标、机构等级判断。必须提供 enumOptions。
2. string：用于诊断名称、机构名称/等级原文、指标值、日期、治疗方案等。
3. 常用 enumOptions：
   - ["已确诊", "未确诊"]
   - ["是", "否", "无法判断"]
   - ["二级及以上", "二级以下", "无法判断"]

# 输出要求

只输出 ruleKeywordGuide JSON 数组，不要 Markdown，不要解释，不要外层包装。

输出结构：

[
  {
    "dataType": "enum|string",
    "required": true,
    "keywordContent": "提取任务描述；必须包含肯定证据、排除边界和优先来源",
    "enumOptions": ["选项1", "选项2"]
  }
]
```

## 四、输出后接哪个节点

后面接代码节点：

```text
节点2：将出参格式化
```

节点2输入绑定：

```text
llm_output = 生成提取项 LLM 节点输出文本
```

## 六、联调注意

1. 腾讯变量不要手打成 DIFY 的 `{{#...#}}` 写法。
2. 如果 LLM 输出带了 ```json 代码块，节点2可以兜底解析，但建议从提示词约束模型直接输出数组。
3. 不要让本节点生成 `keywordCode`。编码通常由上游规则库或后续入库逻辑补齐。
