# 代码节点出入参说明：将出参格式化

对应代码：`代码节点.py`

## 这个节点做什么

把“生成提取项”LLM 节点输出的文本解析成腾讯平台后续节点可直接使用的数组。

兼容以下 LLM 输出形态：

```text
JSON 数组
Markdown 代码块里的 JSON 数组
{"ruleKeywordGuide": [...]}
{"result": [...]}
单个 JSON 对象
```

## 入参

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `llm_output` | str | 是 | 来自节点1 LLM 的输出文本。 |

## 出参

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `result` | [obj] | 解析后的 `ruleKeywordGuide` 数组。解析失败时返回空数组。 |

## 腾讯平台配置

代码节点输入变量：

```text
llm_output = 节点1.Output
类型 = str
```

代码节点输出变量：

```text
result: [obj]
```

结束节点建议绑定：

```text
ruleKeywordGuide = 节点2.result
类型 = [obj]
```

## 可直接粘贴测试的入参示例

```json
{
  "llm_output": "[{\"dataType\":\"enum\",\"required\":true,\"keywordContent\":\"判断材料中是否明确记载2型糖尿病诊断。\",\"enumOptions\":[\"已确诊\",\"未确诊\"]}]"
}
```

## 对应出参示例

```json
{
  "result": [
    {
      "dataType": "enum",
      "required": true,
      "keywordContent": "判断材料中是否明确记载2型糖尿病诊断。",
      "enumOptions": ["已确诊", "未确诊"]
    }
  ]
}
```

## 联调注意

1. 如果 `result` 为空，先检查 `llm_output` 是否实际传到了代码节点。
2. 腾讯平台字段类型建议把 `llm_output` 设为 `str`。
3. 本节点不改写字段含义，只做 JSON 解析和外层拆包。
