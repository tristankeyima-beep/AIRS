# 腾讯智能体平台搭建说明：认定标准提取

本目录用于在腾讯智能体平台复刻 `DIFY工程-认定标准提取` 工作流。

## 一、工作流目标

根据病种名称和单条规则原文，生成后续“材料精解”需要使用的 `ruleKeywordGuide`。

这个流程只负责拆解“应该提取哪些证据”，不判断患者是否符合认定标准。

## 二、主工作流入参

建议在腾讯平台启动节点配置 2 个输入变量：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `diseaseName` | str | 是 | 病种名称，例如 `糖尿病`、`冠心病`。 |
| `ruleContent` | str | 是 | 单条认定标准原文。 |

## 三、节点顺序

```text
启动节点
  -> 节点1：生成提取项（LLM）
  -> 节点2：将出参格式化（代码）
  -> 结束节点
```

## 四、节点配置文件

| 节点 | 类型 | 配置说明 |
| --- | --- | --- |
| 节点1-生成提取项 | LLM | `节点1-生成提取项/【LLM节点配置说明】生成提取项.md` |
| 节点2-将出参格式化 | 代码 | `节点2-将出参格式化/【代码出入参说明】代码节点.md` |

## 五、变量绑定

节点1输入：

```text
diseaseName = 启动节点.diseaseName
ruleContent = 启动节点.ruleContent
```

节点2输入：

```text
llm_output = 节点1.Output
```

如果腾讯平台的 LLM 节点输出字段不是 `Output`，就绑定该节点实际输出的文本字段。

## 六、结束节点出参

建议结束节点输出：

| 字段 | 类型 | 绑定 |
| --- | --- | --- |
| `ruleKeywordGuide` | [obj] | 节点2.result |

## 七、联调注意

1. 节点1只应输出 JSON 数组，不要输出 Markdown 代码块或解释文字。
2. 节点2已兼容 Markdown 代码块、外层对象、`ruleKeywordGuide` 字段和 `result` 字段。
3. 如果最终输出为空数组，优先检查节点1实际输出是否是合法 JSON。
4. `dataType=enum` 时必须带 `enumOptions`；`dataType=string` 时不需要 `enumOptions`。
